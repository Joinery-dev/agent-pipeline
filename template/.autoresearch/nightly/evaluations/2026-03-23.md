## The Ceiling Problem

Every protocol scores 1.000 at baseline. This means:
- The evals cannot distinguish "good" from "great" — they're binary pass/fail checks
- Any mutation that preserves correctness ties the baseline and gets reverted
- Any mutation that breaks something scores below 1.000 and gets reverted
- **The optimizer is trapped**: it can only go sideways or down, never up

## What the Optimizer Tried

Across all protocols, three mutation operators were used:

### `remove-bloat` (tried on all 12 protocols)
Attempted to remove `<personality>` sections, vague checklist items, and redundant subsections. Every time it scored 1.000 — same as baseline — and was reverted. The removed content genuinely appears redundant (behavioral traits already expressed elsewhere), but the evals can't measure the benefit of leaner prompts.

### `add-example` (tried on 5 protocols)
Added concrete examples of desired outputs/behaviors. Same result — 1.000, reverted. The examples were reasonable (e.g., showing success/failure notes for build, before/after fix examples for resolve) but the evals don't check output quality beyond pass/fail.

### `restructure` (tried on 3 protocols)
Reordered steps (e.g., self-review before verify) and merged phases. Same result. Step ordering doesn't matter when the eval only checks end-state.

### Notable failures

- **ship-config iter 1 (prior run)**: LLM wrapped JSON in markdown fences → score 0. Tonight's run avoided this but still reverted for no-improvement.

## Emerging Patterns

1. **The bottleneck has moved from infrastructure to evaluation quality.** Path resolution is fixed. Budget is allocated. The optimizer runs. But it's optimizing against a flat surface.

2. **The optimizer's mutation strategies are sound but unverifiable.** Removing personality sections, adding examples, and restructuring step order are all reasonable optimization moves. The problem isn't the optimizer — it's that the evals are boolean.

3. **Budget fragmentation is less of a concern than it appeared.** Even protocols with 3 iterations (build-basic, build-component, resolve-precision) exhausted useful mutations. More iterations won't help when the eval can't tell good from better.

4. **Consistent pattern: `remove-bloat` → `add-example` → `restructure` → saturated.** Every protocol with 3+ iterations follows this exact sequence. The operator rotation is predictable and has been proven ineffective against 1.000-baseline evals.

## Recommendations

### Priority 1: Harden evaluations before the next nightly run

The single highest-leverage change is making evals more discriminating. Specific suggestions:

- **Fractional scoring**: Replace binary checks with weighted rubrics. E.g., for build: tests pass (0.4) + follows code conventions (0.2) + minimal unnecessary files (0.1) + clean git diff (0.1) + error handling quality (0.2).
- **Adversarial test cases**: Add edge cases that current protocols handle poorly — malformed inputs, ambiguous requirements, conflicting constraints.
- **Quality dimensions**: Check output formatting, explanation clarity, adherence to project conventions, response conciseness.

### Priority 2: Deprioritize mutation runs until evals improve

Running 21 iterations against trivially-satisfied evals burns API budget with zero information gain. Until evals are hardened, set most protocols to baseline-only mode with minimal budget.

### Priority 3: Focus eval hardening on high-impact protocols

Start with `build-basic` and `qa-accuracy` — these are the most exercised protocols in real usage and have the clearest path to multi-dimensional scoring (test quality, convention adherence, false positive/negative rates on harder fixtures).

### Priority 4: Add new mutation operators for post-hardening

Once evals can differentiate, introduce:
- `specialize-for-edge-case`: Target specific eval dimensions that score low
- `tighten-constraint`: Add more specific behavioral constraints
- `remove-ambiguity`: Replace vague instructions with precise ones
- `add-guardrail`: Add checks that prevent specific failure modes
