# Autoresearch Nightly Evaluation — 2026-03-22

## Executive Summary

**Tonight was a near-total loss.** 11 of 12 benchmarks failed at startup due to a **path resolution bug** — they never ran a single iteration. The only benchmark that executed (`ship-config`) was already at a perfect 1.000 baseline and had nowhere to improve.

## Root Cause: Target Path Resolution Bug

The autoresearch runner resolves `targetProtocol` from each benchmark's `meta.json` using three fallback paths in `lib/autoresearch.js:604-615`:

```
1. resolve(targetFile)                                    → CWD/build.md
2. resolve('template', '.claude', 'commands', targetFile) → CWD/template/.claude/commands/build.md
3. resolve('template', '.claude', targetFile)             → CWD/template/.claude/build.md
```

The runner's CWD appears to be `/Users/kevin/Desktop/agent-pipeline/template/`, which means path #2 resolves to `template/template/.claude/commands/build.md` (doubled `template/`). The protocol files actually live at `.claude/commands/*.md` relative to the template directory.

`ship-config.json` was the only target that resolved successfully because it exists at the template root — `resolve('ship-config.json')` from within `template/` finds it directly via path #1.

### Fix options (pick one):
1. **Update meta.json files** to use paths relative to template root: `".claude/commands/build.md"` instead of `"build.md"`
2. **Fix the fallback in autoresearch.js** to try `.claude/commands/` (no `template/` prefix) when CWD is already inside template
3. **Run the nightly from the parent directory** (`agent-pipeline/`) so the `template/` prefix in the fallback is correct

Option 2 is the most robust — it makes path resolution work regardless of CWD.

## ship-config Analysis (the one benchmark that ran)

- **Baseline: 1.000** — already perfect. All 5 range-checked values (`maxQARounds`, `maxPMReplans`, `maxBuildBatch`, `progressCheckIntervalMs`, `maxChecksWithoutProgress`) were within acceptable ranges.
- **Iteration 1 (remove-bloat):** Removed `heartbeatIntervalMs`. Score: 1.000 (no improvement) → reverted. Correct decision — removing an observability field doesn't improve the eval score.
- **Iteration 2 (add-example):** Changed `progressCheckIntervalMs` and `maxChecksWithoutProgress` to mid-range values. **Failed catastrophically** — the LLM wrapped its JSON output in markdown code fences (`` ```json ... ``` ``), producing invalid JSON → score 0.000 → reverted.
- **Budget exhausted** after 2 iterations (budget was only 0.042 hours ≈ 2.5 minutes).

### LLM Optimizer Struggles

1. **Markdown wrapping of JSON** — classic LLM failure mode. The iter-002 revert file at `history/iter-002-revert.md` confirms the output was wrapped in `` ```json `` fences. The eval correctly rejected it (`"Invalid JSON: Unexpected token"`).
2. **Optimizing a perfect score** — when baseline is 1.000, the optimizer has no gradient signal. Every mutation either matches (revert, no improvement) or regresses. This benchmark should be deprioritized.
3. **Tiny budget** — 2.5 minutes per protocol meant ship-config only got 2 iterations, and the other protocols got 0.

## Emerging Patterns

1. **Infrastructure before optimization.** No amount of budget allocation matters if the targets can't be found. Fix the path bug first.
2. **Budget fragmentation.** Splitting budget equally across 12 benchmarks (~2.5 min each) means no protocol gets enough iterations to learn. Concentrate budget on fewer protocols per night.
3. **No baseline data for 11 protocols.** We have zero signal on which protocols are weak. The first priority after fixing paths is establishing baselines.

## Recommendations

1. **Fix the path resolution bug before the next nightly run.** Nothing else matters until this works.
2. **On the first successful night, run baseline-only** for all protocols to establish scores without spending budget on mutations.
3. **For ship-config:** Add JSON-format enforcement to the mutation prompt (e.g., "Output raw JSON only, no markdown fences"). Consider marking it as low-priority since baseline is already 1.000.
4. **Budget strategy:** After baselines are established, allocate 70% of budget to the 3-4 lowest-scoring protocols and split the remaining 30% across the rest.
